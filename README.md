# WorkBuddy 每日签到技能

## 简介

本技能用于自动化执行 WorkBuddy 应用程序的每日签到操作，通过模拟用户点击来完成签到领取积分的流程。

## 功能特性

- 自动检测并启动 WorkBuddy（如程序未运行）
- 自动点击用户菜单并完成签到
- 支持定时自动签到（每天 09:00）
- 包含 PowerShell 和 Shell 两种脚本版本

## 使用方法

### 前提条件

- 安装 Playwright CLI 工具
- WorkBuddy 程序已配置 QQ 浏览器内核
- 屏幕分辨率建议 1920x1080 或更高

### 手动执行

**PowerShell（Windows）：**
```powershell
.\scripts\checkin.ps1
```

**Shell（Linux/macOS）：**
```bash
bash scripts/workbuddy_checkin.sh
```

### 自动签到配置

本技能已配置自动化任务：
- **任务名称**：WorkBuddy每日签到
- **执行时间**：每天 09:00
- **状态**：ACTIVE

## 签到流程

1. 检测 WorkBuddy 是否正在运行，如未运行则启动
2. 等待程序加载完成（约 3-5 秒）
3. 点击左下角用户菜单（CSS: `.user-menu`）
4. 等待菜单展开（约 2 秒）
5. 点击「立即领取 →」按钮（CSS: `.daily-checkin-banner-action`）
6. 完成签到并验证积分到账

## 注意事项

1. 每步操作之间建议有适当等待时间，确保界面渲染完成
2. 若某步骤失败，应记录日志并重试（最多 3 次）
3. 建议在无人值守环境下运行，避免鼠标键盘冲突
4. DOM 结构可能随版本更新变化，需定期验证 CSS Selector
5. 确保 WorkBuddy 窗口在主显示器上可见

## 故障排除

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 找不到用户菜单 | 界面未完全加载 | 增加等待时间至 5 秒 |
| 签到按钮不可见 | 菜单未展开 | 确保先点击用户菜单 |
| 点击无效 | 元素被遮挡 | 确保 WorkBuddy 窗口在最前 |
| 程序启动失败 | 路径错误 | 检查 WorkBuddy 安装路径 |
